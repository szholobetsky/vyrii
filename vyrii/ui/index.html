svitovyd

Project map — index code structure, then find/trace/deps/sym/keywords.

Scan your code and build a map of all functions, classes, and their links. Then explore the structure.
Scan the project folder. Find all function and class definitions. Save links between them. Result: .svitovyd/map.txt.
2 — definitions and calls. 3 — also variables and parameters. Use 2 for most projects.
Filter the map by keyword. Examples: auth — find blocks with "auth" auth !test — find "auth", skip "test" \insertUser — find exact identifier "insertUser"
Pick one function or class name. See who calls it. Useful when you want to know: what breaks if I change this?
Pick one function or class name. See what it calls. Useful when you want to know: what does this depend on?
Find functions that many others call, but which call very few. These are often good candidates for refactoring.
No task text — list the most used identifiers in this codebase. With task text — find identifiers related to that text.
Split camelCase and snake_case. Finds more matches.
Compare two map snapshots. Shows what functions and files changed. Useful after a big refactor.
Copy .svitovyd/map.txt to another name before a refactor, then compare after.
